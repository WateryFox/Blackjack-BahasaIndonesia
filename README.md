I built this project to create a fully functional Blackjack game that runs entirely in the command line. Unlike a simple text game, this version includes actual casino mechanics like betting systems, card shuffling, and specialized moves like doubling down or splitting pairs. My goal was to take the basic rules of the game and translate them into a logical flow where the computer acts as the dealer and the player has to manage their virtual bankroll. I wanted to see if I could recreate the tension of a real card game using nothing but Python logic and terminal output.

The core of the game relies on a deck generation system that uses dictionaries to store the suit and value of every card. I implemented a custom shuffling algorithm using the random module to ensure every round feels different. One of the most interesting parts to code was the logic for the Ace card because its value has to change from eleven to one depending on whether the player is about to bust. I also spent a lot of time on the dealer's behavior, writing a loop that forces the computer to keep hitting until it reaches a minimum score of seventeen, which mimics how real dealers are required to play in a casino.

I made sure the game included more than just hitting and standing. I added a split mechanic that triggers if you receive two cards of the same value, effectively allowing you to play two separate hands at once. The game also tracks your total money across multiple rounds, meaning you can actually go bankrupt if you aren't careful with your bets. I used the time module to add small delays when the dealer is drawing cards to make the experience feel more realistic and less like the computer is just instantly spitting out results. It was a great exercise in managing complex conditional statements and keeping the game state organized.

If I were to expand on this project, I would love to add a multiplayer mode where two people can play against the dealer at the same time. I am also interested in implementing a save system so players can come back later and continue with the same amount of money they had when they left. Another cool addition would be a basic graphical interface using a library like Pygame to replace the text based cards with actual images, though I really enjoyed the challenge of making the terminal version look as clean as possible using symbols and formatted strings.
